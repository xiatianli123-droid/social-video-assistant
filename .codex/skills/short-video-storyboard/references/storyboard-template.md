# 短视频分镜交付模板

## Brief Intake

如果用户只给了产品名或 brief 不完整，先打开 `outputs/storyboard-brief-selector.html`，请用户填写并把右侧生成的简报发回；不要直接生成完整分镜。只有用户明确要求“你来定”“快速出一版”“直接生成”或“按默认信息先写”时，才可以基于合理假设继续，并在创意方向中标注“默认假设”。

## 分镜表

必须使用逐镜头块格式，不使用宽 Markdown 表格，也不要输出只有“镜头/时长/广告功能/核心画面”的摘要表。每个镜头字段顺序固定为：`镜头 N`、`时长：`、`景别：`、`运镜：`、`运镜逻辑：`、`画面描述：`、`文案旁白：`、`音效配乐：`。`时长` 必须写累计时间段，例如 `0-2.5秒`、`2.5-5秒`、`5-8秒`。

文案旁白规则：

- `文案旁白` 默认使用英文旁白或英文人物对话，除非用户明确要求中文文案。
- 有声旁白写在同一字段内：`音色：[定制音色说明]“English line”`。
- 同一条旁白声线在全片重复同一音色锚点；同一个人物说话保持同一人物声音设定。
- 不使用屏幕文案，除非用户明确要求字幕/屏显文字。

音效配乐规则：

- `音效配乐` 保持单字段，根据画面主体需要写几个类别就写几个，尽量精简。
- 全景环境可只写 `环境声`；人物对话写 `人声`；动作或道具成为主体时写 `动作音效` / `道具声`；`BGM` 只在需要情绪铺底或节奏推进时写。
- 音效必须服务画面主体表达，并来自当前画面、人物动作、道具、场景或剪辑节奏。

经验参考：

- 生活方式产品可参考 `references/lifestyle-product-storyboard.md` 的抽象经验：先建立体验缺口和产品在场，再写产品介入瞬间，用人物反应、环境反馈或道具变化证明卖点，最后把产品和用户结果绑定成记忆点。

## 设计解析

- 景别逻辑：远/全景建立环境，中景推进动作，近景/特写承载情绪或产品证明。
- 运镜逻辑：每个镜头必须说明运镜目的。固定镜头稳定信息，推镜头聚焦卖点，摇/移镜头揭示空间或引导视线，跟拍增强临场感，拉镜头释放空间和展示结果，环绕镜头强化主体记忆。
- 短视频节奏：每个镜头都要说明它的广告功能；如果前 0.5 秒看不出卖点或问题，就改成触发点/结果瞬间。
- 分镜连续性：人物身份、服装、场景结构、产品位置、光线方向和色彩风格在各镜头之间保持一致。
- 情绪曲线：触发前需求 -> 产品介入瞬间 -> 体验结果/人物反应 -> 产品或品牌记忆。
- 格式完整性：不得把逐镜头块压缩成四列摘要表；不得添加 `执行备注`、`首帧图方向`、`首帧方向`；如需写 match cut 或转场链路，必须服务镜头承接，不能替代完整镜头字段。

## HTML 输出要求

- 单文件 HTML，可直接在浏览器打开。
- 使用内联 CSS，不依赖外部 CDN。
- 包含项目概览、分镜表和设计解析。
- 只有已生成首帧图片时，才使用本地相对路径展示图片卡片；未生成图片时不展示首帧卡片。
