# 短视频分镜交付模板

## 生成前提问卡

如果用户只给了产品名，或关键信息少于 3 项，先问这 6 个问题，不要直接生成完整分镜：

1. 产品/品牌是什么？
2. 想要的视频风格？（现代 / 温馨 / 活力 / 专业 / 高级 / 科技 / 其他）
3. 镜头重点放在哪里？（产品 / 人物 / 使用体验 / 场景氛围 / 前后对比）
4. 希望观众产生什么情绪？（信任 / 温馨 / 惊喜 / 清爽 / 高级感 / 购买欲）
5. 场景设定在哪里？（家居 / 户外 / 商业空间 / 工作室 / 其他）
6. 有没有一句话大纲或必须出现的画面？

如果用户说“你来定”“快速出一版”“直接生成”，可以基于合理假设继续，但要在创意方向中标注“默认假设”。

文案旁白规则：

- `文案旁白` 默认使用英文旁白或英文人物对话，除非用户明确要求中文文案。
- 有声旁白写在同一字段内：`音色：[定制音色说明]“English line”`。
- 同一条旁白声线在全片重复同一音色锚点；同一个人物说话保持同一人物声音设定。
- 不使用屏幕文案，除非用户明确要求字幕/屏显文字。

音效配乐规则：

- `音效配乐` 保持单字段，根据画面主体需要写几个类别就写几个，尽量精简。
- 全景环境可只写 `环境声`；人物对话写 `人声`；动作或道具成为主体时写 `动作音效` / `道具声`；`BGM` 只在需要情绪铺底或节奏推进时写。
- 音效必须服务画面主体表达，并来自当前画面、人物动作、道具、场景或剪辑节奏。

案例参考：

- 生活方式产品、庭院/泳池/派对场景、低噪/大风量/氛围卖点，可参考 `references/case-pool-party-fan-light.md` 的结构：先用痛点/场景和产品在场做 hook，再直接切到风量触发点，用人物正常音量交流证明低噪，最后回到产品记忆。

## 设计解析

- 景别逻辑：远/全景建立环境，中景推进动作，近景/特写承载情绪或产品证明。
- 运镜逻辑：每个镜头必须说明运镜目的。固定镜头稳定信息，推镜头聚焦卖点，摇/移镜头揭示空间或引导视线，跟拍增强临场感，拉镜头释放空间和展示结果，环绕镜头强化主体记忆。
- 短视频节奏：每个镜头都要说明它的广告功能；如果前 0.5 秒看不出卖点或问题，就改成触发点/结果瞬间。
- 分镜连续性：人物身份、服装、场景结构、产品位置、光线方向和色彩风格在各镜头之间保持一致。
- 情绪曲线：触发前需求 -> 产品介入瞬间 -> 体验结果/人物反应 -> 产品或品牌记忆。

## AIG 首帧生成规则

- 首帧图按顺序生成，不批量并行生成。
- Shot 1 先生成场景基准图，确认空间结构、人物/产品、色彩和光线。
- Shot 2 使用 Shot 1 作为参考图生成，只改变剧情需要改变的动作、光线或镜头距离。
- Shot 3 使用 Shot 2 作为身份/场景参考，但不能复制构图；必须优先匹配“行动推进、照明覆盖、使用结果”等当前分镜任务。
- Shot 4 使用上一张可接受图片和产品参考作为锚点，但不能复制构图；必须优先匹配产品特写、结果收束、证据画面或品牌记忆等当前分镜任务。
- 每张首帧都应该已经进入动作或结果：不要把首帧放在人物走过去、刚要看见、刚要递出、镜头刚要上仰之前。
- AIG image prompt 是内部执行信息，不在最终表格或 HTML 中展示。

## HTML 输出要求

- 单文件 HTML，可直接在浏览器打开。
- 使用内联 CSS，不依赖外部 CDN。
- 包含项目概览、分镜表和设计解析。
- 只有已生成首帧图片时，才使用本地相对路径展示图片卡片；未生成图片时不展示首帧卡片。
