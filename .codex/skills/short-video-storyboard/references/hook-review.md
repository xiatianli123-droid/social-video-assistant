# Hook 审查参考

这个文件保存短视频分镜的 Hook 审查规则和可复用案例库。写分镜、改分镜、专门审查开头钩子或判断首镜强弱时读取本文件。

## 使用方式

- 分镜完成后、输出给用户前，先内部审查首镜和整体开场。
- 不要把审查问题、Hook 等级或评分写进最终交付；最终只输出分镜字段里的画面内容、运镜逻辑、文案和声音。
- 如果首镜偏弱，先重写首镜，再交付。
- 每次完成 Hook 审查或 Hook 改写后，把本次案例追加到本文末尾的 `新增案例库`。记录要短，优先保留可复用判断经验。

## 四问审查

内部逐项回答 4 个问题：

- 首帧事件：第一帧是否已经有事件发生，而不是只建立环境？
- 0.5 秒可见冲突：不听旁白时，观众是否能看见异常、冲突、反差、危险、结果悬念或明确痛点？
- 观众继续看的问题：画面是否会自然制造“为什么 / 接下来怎样 / 怎么解决 / 这到底是什么”的问题感？
- 产品如何回答这个问题：后续产品介入是否直接解决或反转开场问题，而不是靠无关猎奇吸引注意？

## 判定规则

- 4 项都成立，才可视为真强 Hook。
- 3 项成立且产品因果清楚，可视为有效但中等 Hook。
- 少于 3 项成立，或首镜只是普通状态描述、氛围铺垫、偏暗、闷热、冷清、缺少氛围，必须改写首镜。
- 不要把“钩子设计：强 Hook”当成结论；只看画面机制是否成立。

## 改写规则

如果首镜偏弱，优先把“状态描述”改成“正在发生的结果”。

- 不是“客厅偏暗”，而是孩子已经把颜色画错、母亲看不清书页、客人避开餐桌、关键物件落入暗处、危险/尴尬/错失正在发生。
- 不是“庭院闷热”，而是餐巾已经贴到汗湿手臂、饮料冰块快速融化、客人正在撤向室内、桌边座位已经空掉。
- 不是“空间缺少氛围”，而是某个社交动作已经失败：合影拍糊、餐桌没人停留、关键装饰被忽略、人物沟通被环境打断。

## 基础案例

- 真强 Hook 正面：跳伞冲出舱门，第一帧身体已经离开安全空间，海岸线翻转，观众立刻想知道会怎样落地或转场。
- 真强 Hook 正面：城市激光危机，红眼激光切开楼体并逼近家庭窗口，观众立刻想知道这户人家如何逃脱或反击。
- 有效但中等 Hook 正面：泳池派对流失，餐桌没人坐、朋友挤向室内冷气，问题可见，产品后续把人拉回户外。
- 有效但中等 Hook 正面：耳钉掉进暗角，小物件丢失制造轻悬念，产品后续用下照找回细节。
- 有效但中等 Hook 正面：双管灯光剑误导，观众先把双管光源误认为科幻武器，后续反转为车间灯卖点。
- 弱 Hook 反面：普通客厅偏暗，只是在说明照明不足，没有强事件、异常或继续看的问题。
- 弱 Hook 反面：风扇灯闷热场景，只是人物坐着觉得热，冲突轻且画面停留理由不足。
- 弱 Hook 反面：灯串庭院昏暗，只是在建立氛围缺失，产品虽能点亮空间，但首镜缺少事件和问题感。

## 新增案例记录格式

每次完成 Hook 审查或 Hook 改写后，在 `新增案例库` 末尾追加一条。保持短条目，不要粘贴完整分镜。

```markdown
- 日期：YYYY-MM-DD
  项目/产品：
  原始开场：
  审查结论：
  最终处理：
  可复用经验：
```

## 案例库维护规则

- `新增案例库` 超过 20 条后必须整理一次。
- 每类 Hook 问题最多保留 3-5 条代表案例；重复判断逻辑的案例要合并。
- 没有可复用经验、只记录项目细节的案例要删除。
- 连续出现 3 次的判断经验应提炼进 `判定规则` 或 `改写规则`，再删除多余案例，只保留 1 条代表。
- 主文件只保留当前最有教学价值的精选案例；历史材料如需保留，移入 `references/archive/hook-review-cases-YYYY.md`。
- 归档文件不默认读取；只有用户要求追溯历史案例，或需要复盘旧判断时才读取。
