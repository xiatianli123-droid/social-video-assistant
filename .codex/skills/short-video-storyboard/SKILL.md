# 短视频分镜脚本

为品牌推广短片生成可拍摄、可提案、可交给 AI 视频工具执行的中文分镜脚本。这个 skill 只负责分镜脚本、连续性检查和 HTML 交付衔接；不直接生成首帧图片。

## 硬性流程

1. 先判断 brief 是否完整。如果用户只给了产品名，或少于 3 个简短字段，先问卷或打开 `outputs/storyboard-brief-selector.html`，并等待用户把简报发回；此时不要写分镜、不要创建项目输出目录、不要生成 HTML，除非用户明确要求“按默认信息先写”。
2. brief 完整后，先完成中文分镜脚本。
3. 每个镜头只写分镜表字段，不执行图片生成。
4. 分镜完成后必须生成或更新最终 HTML；使用或遵循 `storyboard-html-delivery`。
5. HTML 完成后，在会话里直接询问首帧确认，不使用 HTML 确认面板。
6. 只有用户在会话中明确确认“需要生成首帧图”后，才使用 `storyboard-first-frame-images`。
7. 如果用户选择“暂不生成首帧图”，流程也视为完成；HTML 仍然是最终交付。

## 需求收集

生成分镜前，确认以下信息；如果用户已经提供，就不要重复追问：

- 产品 / 品牌 / 服务
- 视频风格：现代、温馨、活力、专业、高级、科技、文学感、玩趣等
- 镜头重点：产品、人物、使用体验、场景氛围、痛点对比、品牌调性
- 目标情绪：信任、温暖、惊喜、清爽、高级感、购买欲、安心、向往等
- 场景位置：家居、户外、办公室、商店、泳池、街道、棚拍或具体环境
- 一句话分镜方向或大致创意
- 目标受众和投放平台
- 核心卖点
- 人物设定：年龄、服装、发型、角色、表情、动作连续性

如果用户只给了产品名，或少于 3 个简短字段，先问一个简洁问卷。一次最多问 6 个问题，优先确认：产品/服务、视觉风格、镜头重点、目标情绪、场景、一句话方向。

如果用户要求按钮、点击选择或更快的需求确认，复用 `outputs/storyboard-brief-selector.html`，让用户把右侧生成的简报发回会话。不要为每个视频项目复制或新建按钮 HTML。

## 默认中文问卷

```markdown
为了把分镜方向定准，我先确认 6 个信息：

1. 产品 / 品牌是什么？
2. 想要的视频风格？（现代 / 温馨 / 活力 / 专业 / 高级 / 科技 / 其他）
3. 镜头重点放在哪里？（产品 / 人物 / 使用体验 / 场景氛围 / 前后对比）
4. 希望观众产生什么情绪？（信任 / 温暖 / 惊喜 / 清爽 / 高级感 / 购买欲）
5. 场景设定在哪里？（家居 / 户外 / 商业空间 / 工作室 / 其他）
6. 有没有一句话大纲或必须出现的画面？
```

## 镜头设计原则

- 用远景或环境镜头建立场景。
- 用中景表现人物动作、产品关系和使用情境。
- 用近景或特写强调情绪、功能证明和记忆点。
- 运镜必须服务叙事：揭示信息、引导视线、跟随动作、增强情绪或释放空间。
- 每个镜头都要写清“运镜逻辑”，说明这个运动为什么存在。
- 情绪从痛点、紧张或好奇，推进到释放、信任、愉悦、安心或向往。

## 短视频节奏原则

短视频分镜不能像长视频截取出来的一段。每个镜头都必须在开头就提供新的广告信息，让观众在前 0.5 秒知道这个镜头在卖什么。

- 每个镜头先写清一个广告功能，再写画面：1 秒内建立痛点/场景、立即展示核心卖点、人物反应证明卖点、产品记忆点/购买理由。
- 少写完整过程，多写触发点和结果瞬间。优先使用“触发前：需要产品 -> 触发瞬间：产品介入 -> 结果瞬间：人舒服/场景变好 -> 记忆瞬间：产品与场景绑定”的结构。
- 避免镜头主要内容只是“人物走过去”“看见对方”“慢慢靠近”。人物动作只是载体，卖点变化才是目的。
- 每个镜头首帧必须已经进入动作：例如餐巾已经被风掀起、女生发丝已经动了、饮品已经递到手边、两人已经对视微笑、风扇灯已经是主体。
- 15 秒内不要承载太多人物关系。浪漫、家庭感、社交感或亲密感只能作为氛围，必须服务卖点证明：微笑=舒适，正常音量说话=低噪，发丝/餐巾动=大风量，大家留在户外=场景适配。
- 用剪辑跳跃替代真实走位。空间、人物、道具、产品状态、光线和色彩连续即可，不必展示每一步走路、抬头、靠近或上仰过程。
- 如果某个镜头说不出广告功能，或前 0.5 秒看不出它在卖什么，必须重写。

## Hook 审查机制

写出“强 hook”不等于真的有 hook。分镜完成后、输出给用户前，必须在内部审查首镜和整体开场，不要把审查问题、hook 等级或评分写进最终交付；最终只输出分镜字段里的画面内容、运镜逻辑、文案和声音。

内部逐项回答 4 个问题：

- 首帧事件：第一帧是否已经有事件发生，而不是只建立环境？
- 0.5 秒可见冲突：不听旁白时，观众是否能看见异常、冲突、反差、危险、结果悬念或明确痛点？
- 观众继续看的问题：画面是否会自然制造“为什么 / 接下来怎样 / 怎么解决 / 这到底是什么”的问题感？
- 产品如何回答这个问题：后续产品介入是否直接解决或反转开场问题，而不是靠无关猎奇吸引注意？

审查规则：

- 4 项都成立，才可视为真强 hook。
- 3 项成立且产品因果清楚，可视为有效但中等 hook。
- 少于 3 项成立，或首镜只是“普通场景偏暗、闷热、冷清、缺少氛围”，必须改写首镜。
- 不要把“钩子设计：强 hook”当成结论；只看画面机制是否成立。

Hook 等级参考：

- 真强 hook 正面：跳伞冲出舱门，第一帧身体已经离开安全空间，海岸线翻转，观众立刻想知道会怎样落地或转场。
- 真强 hook 正面：城市激光危机，红眼激光切开楼体并逼近家庭窗口，观众立刻想知道这户人家如何逃脱或反击。
- 有效但中等 hook 正面：泳池派对流失，餐桌没人坐、朋友挤向室内冷气，问题可见，产品后续把人拉回户外。
- 有效但中等 hook 正面：耳钉掉进暗角，小物件丢失制造轻悬念，产品后续用下照找回细节。
- 有效但中等 hook 正面：双管灯光剑误导，观众先把双管光源误认为科幻武器，后续反转为车间灯卖点。
- 弱 hook 反面：普通客厅偏暗，只是在说明照明不足，没有强事件、异常或继续看的问题。
- 弱 hook 反面：风扇灯闷热场景，只是人物坐着觉得热，冲突轻且画面停留理由不足。
- 弱 hook 反面：灯串庭院昏暗，只是在建立氛围缺失，产品虽能点亮空间，但首镜缺少事件和问题感。

如果首镜偏弱，优先把“状态描述”改成“正在发生的结果”：不是“客厅偏暗”，而是孩子已经把颜色画错、母亲看不清书页、客人避开餐桌、关键物件落入暗处、危险/尴尬/错失正在发生。

## 物理世界连续性

分镜设计必须先通过连续性检查：

- 天气、时间、空间、人物、服装、道具、产品状态、光线、色彩系统必须连续。
- 每个镜头新增的视觉元素必须有来源：前面已经埋过、由当前动作产生，或符合场景常识。
- 卖点证明不能压过世界状态。当前 3 镜是干燥庭院时，第 4 镜不能突然出现水滴；如果要表现防水，必须从前面就设定雨后、喷淋、泳池边、浇花或其他合理水源。
- 当卖点和连续性冲突时，优先保证连续性，再换卖点表达方式，或把卖点移动到另一个脚本版本。

### 文案旁白写法

- `文案旁白` 默认写英文旁白或英文人物对话，除非用户明确要求中文文案；分镜字段、创意方向和设计解析仍用中文。
- 把英文旁白/人物对话和音色写在同一个 `文案旁白` 字段里，不要另拆“英文旁白/屏幕文案”和“旁白音色说明”。
- 默认不写屏幕文案；只有用户明确要求屏显文字或字幕时才加入。
- 无旁白镜头写：`（无声）`。
- 音色必须根据产品、场景、视觉风格、目标情绪和文案内容定制，不要固定套用某个示例音色。
- 有旁白镜头写：`音色：[根据本片内容定制的音色说明]“English voiceover line”`。
- 同一个视频里的旁白有声镜头要重复同一条定制音色锚点；同一个人物的对话也保持同一人物声音设定，保证配音执行时音色连续。

### 音效配乐写法

- `音效配乐` 必须服务当前分镜的画面主体表达和剪辑节奏，而不是为了凑齐声音分类。
- 保持一个 `音效配乐` 字段，根据画面需要写几个类别就写几个，尽量精简：全景环境可只写 `环境声`；人物对话写 `人声`；动作或道具成为主体时写 `动作音效` / `道具声`；`BGM` 只在需要情绪铺底或节奏推进时写。
- 只写需要被听到的声音。不要在音效栏写“不加某声音 / 不添加某音效 / 不额外添加某产品声”这类反向说明；没有出现的声音直接省略。
- 产品如果在真实世界里不该有明显声音，就不要写产品音效。低噪、静音、柔和类卖点用画面里的环境声、人物反应、正常交流音量来侧面证明。
- 音效必须有画面来源：来自场景、人物动作、道具、产品状态变化或剪辑转场。不要为了证明卖点凭空添加无来源声音。
- BGM 氛围要写清情绪、节奏和乐器/质感，例如温暖干净的现代家居轻音乐、轻微钢琴、柔和合成器、低频铺底。
- 如果当前镜头只需要安静氛围，就只写实际存在的环境底噪或 BGM；如果完全不需要声音，写“（无）”。

## HTML 与首帧确认

- 分镜脚本完成后，必须使用或遵循 `storyboard-html-delivery` 创建 `storyboard.html`。
- 如果尚未生成首帧图，HTML 不展示图片卡片。
- HTML 完成后，在会话里直接询问用户是否需要生成首帧图，提供三个选项：“需要生成首帧图 / 暂不生成首帧图 / 先修改分镜再决定”。
- 不使用首帧确认 HTML 面板。
- 如果用户确认需要首帧图，再使用 `storyboard-first-frame-images`。

## 质量标准

- 每个镜头必须有不同作用，避免 4 个镜头都只是产品美图。
- 每个镜头的景别、运镜、构图都要解释清楚它如何服务情绪或卖点。
- 每个镜头时长必须匹配信息量和广告功能；短镜头要清楚，长镜头不能空转。
- 每个镜头必须通过短视频节奏检查：开头已经在动作中，前 0.5 秒能看出广告功能，不用完整人物走位填时长。
- 每次分镜完成后必须通过 Hook 审查机制；如果首镜只是弱 hook 或痛点铺垫，先重写首镜再交付。
- 卖点尽量用画面证明，而不是靠抽象文案解释。
- 文案要匹配所选视频时长和单镜时长。短镜头只适合极短英文旁白或短角色对白；不要因为镜头变长而塞满旁白。
- 保持同一空间关系、同一产品、同一人物、同一服装、同一光线逻辑。
- 结尾必须形成产品或品牌记忆：可以用产品定格、包装、应用画面、前后对比、情绪收口或用户结果来完成；只有用户明确提供时，才加入 logo、slogan、CTA 或合规限制。
